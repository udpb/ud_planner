# Ch.5 자주 빠지는 함정

> 이 챕터를 읽으면 선배들이 이미 밟아 본 지뢰 7개의 위치를 알게 됩니다. 나는 밟지 말아야 합니다.

## 함정 1. Value Chain 없이 장표부터 쓰기

**원칙**: 5계층 인과 사슬 (Input → Activity → Output → Outcome → Impact) 을 한 장으로 확정한 뒤에 장표를 연다.

### 왜 이 순서인가

제안서의 섹션 배치 · 페이지 분량 · 배점 매핑은 모두 Value Chain 에서 파생됩니다. Value Chain 이 없으면 각 섹션이 독립된 조각처럼 쓰여, 평가위원이 "이 Activity 는 어떤 Outcome 과 연결되는가" 를 스스로 재구성하며 읽어야 합니다. 재구성 부담이 오는 순간 점수가 깎입니다.

### 이걸 어기면 나오는 증상

- 피드백 한 번에 섹션 순서·논리 구조가 뒤집힘
- 페이지 분량은 늘어나는데 "그래서 뭐?" 에 답이 없음
- 같은 Activity 를 팀원마다 다른 맥락으로 서술
- 마감 직전까지 구조 수정이 이어져 오타·비문이 남음

### 대응 — 이 순서대로

```
1. RFP 정밀 분석 (Ch.3)
       │
       ▼
2. 5계층 Value Chain 한 장 확정 (Ch.4)
       │     ← 이 지점 전에 장표 작업 금지
       ▼
3. 팀 공유 · 본부장 승인
       │
       ▼
4. 섹션별 장표 작성 (Ch.6)
```

### 체크포인트

- [ ] 5계층이 A4 한 장에 모두 들어갔는가
- [ ] 계층 사이 "그래서?" 테스트를 모두 통과했는가
- [ ] 이 한 장을 팀 전체(본부장 포함)가 같이 보고 합의했는가
- [ ] 합의 이후에만 장표 작업을 시작했는가

> **참고 사례**: [코오롱 프로보노](../03-casebook/06-startup-education/kolon-probono.md) — Value Chain 없이 장표부터 쓰다 2주 무한수정. 중간 리셋 후 반나절 Value Chain 확정 → 1.5일 완성, 품질도 개선.

## 함정 2. 이론 나열, 현장 실행 요소 누락

**원칙**: 사업 유형과 무관하게 제안서는 **"수혜자가 직접 해 보는 구간"** 이 드러나야 한다. 이론·강의만 연속되면 실행력 부족 신호로 읽힌다.

### 왜 이 원칙이 사업 유형을 가리지 않는가

평가위원의 공통 질문은 **"이 사업이 끝났을 때 수혜자에게 실제로 무엇이 남는가"** 입니다. 남는 것은 이론 요약본이 아니라 **현장에서 해 본 경험·결과물·데이터** 입니다. 사업 유형마다 "해 본다" 의 모습은 다르지만 원칙은 같습니다.

### 사업 유형별 "해 본다" 의 형태

| 사업 유형 | 실행 구간의 형태 |
|---|---|
| 창업 교육 | Action Week — 시제품 테스트·현장 판매·고객 인터뷰 |
| 소상공인 지원 | 매장 리뉴얼 현장 적용·실매출 테스트·고객 반응 수집 |
| 로컬 상권 | 팝업 운영·브랜딩 쇼케이스·상권 내 협업 실행 |
| 공모전·디자인 | 샘플 실물 제작·유통 입점 테스트·대중 반응 수집 |
| 문화·장인 | 공동 창작 세션·현장 시연·교류 참여 |
| 매칭 (멘토링·프로보노) | 멘토-수혜자 공동 프로젝트·중간 점검·결과 발표 |
| 글로벌 진출 | 해외 현지 방문·바이어 미팅·MVP 현지 테스트 |

### 이걸 어기면 나오는 증상

- 이론 3회 연속 → 참여자 집중도 저하, 발주처 평가에서 "실행력 부족" 신호
- 커리큘럼의 "강의·특강" 비중 50%+, "적용·실행" 10% 미만
- 산출물이 "수료증·보고서" 에 그쳐 Outcome 수치로 연결되지 않음

### 대응

- 이론 3회 연속 구간이 잡히면 **사업 유형에 맞는 "해 보는 구간"** 을 1회 이상 삽입
- 커리큘럼 표에 **이론 / 적용 / 실행** 3분류 비율을 사전 설정 (권장: 이론 30~40%, 적용·실행 60~70%)
- 실행 구간의 산출물을 Output·Outcome 과 연결

### 체크포인트

- [ ] 커리큘럼에 이론 3회 연속 구간이 없는가
- [ ] 사업 유형에 맞는 "해 보는 구간" 이 명시적으로 표기되었는가
- [ ] 실행 구간의 결과물이 평가 배점 항목과 연결되는가

> 참고: 언더독스 실행 철학 **"해 보기 전엔 아무것도 모른다"** ([부록 B](../appendix/b-ud-assets.md)).

## 함정 3. 모호한 수량 표현

"많은 전문가", "다양한 프로그램", "충분한 지원", "최적의 환경" — 이런 표현은 평가위원에게 **"수치로 말할 자료가 없다"** 는 신호입니다. 정량 포화 원칙을 위배합니다.

### 교체 예시

| 모호한 표현 | 정량 포화 표현 |
|-------------|--------------|
| 많은 코치진 | 전국 800명 코치 풀, 261명 액션코치 (사례별 수치는 원본 확인) |
| 다양한 프로그램 | 주차별 5개 프로그램, 총 8주 40개 세션 |
| 충분한 지원 | 1:1 코칭 주 1회, 전문 멘토 월 1회, 동료 네트워크 상시 |
| 풍부한 네트워크 | 520 글로벌 파트너, 93개 시·군·구 사업 수행 이력 |

**대응**: 초안 완성 후 `많은` `다양한` `충분한` `풍부한` `최적의` 이 다섯 단어를 Ctrl+F 로 검색. 나온 곳마다 숫자로 바꾸거나 문장 삭제.

## 함정 4. 예산만 보고 커리큘럼 대충 짜기

RFP 에는 항상 평가 배점이 있습니다. 배점 분포를 무시하고 "예산 규모에 맞게 적당히" 커리큘럼을 짜면, 최고 배점 항목에 페이지를 적게 할애하는 실수가 나옵니다.

### 배점 역배분의 전형 (실패 예)

- 평가 기준: 수행 계획 40점, 기대 효과 30점, 제안사 역량 20점, 예산 10점
- 제안서: 제안사 소개 8페이지, 수행 계획 5페이지, 기대 효과 2페이지, 예산 1페이지
- **결과**: 최고 배점 항목 (수행 계획) 이 가장 얇음 → 배점 감점 20~30점.

### 올바른 분포 (성공 예)

- 같은 배점일 때
- 수행 계획 14페이지, 기대 효과 8페이지, 제안사 소개 4페이지, 예산 1페이지 + Section V 보너스 1페이지

**대응**: 초안 쓰기 전 평가 배점 표를 만들고 "각 항목별 페이지 수 분포" 를 대략 계산. 작성 후 실제 분포와 목표 분포를 비교.

## 함정 5. 서포터에게 방향 없이 리서치 시키기

제안서 작성에 서포터 (주니어·인턴·외주 리서처) 를 투입할 때, "경쟁 사업 리서치 부탁해" 같이 방향 없이 맡기면 결과물이 쓸모없거나 재작업이 됩니다.

### 리서치 의뢰의 나쁜 예 vs 좋은 예

- **나쁜 예**: "유사 사업 사례 조사해 주세요"
- **좋은 예**: "경기도·서울시·부산시의 2023~2025년 로컬 브랜드 사업 중, 예산 3억 이상·기간 6개월 이상 사업 3건을 찾아서 각 사업의 (1) 핵심 메시지 (2) Value Chain 5계층 (3) 차별화 포인트를 A4 1장 × 3건으로 정리해 주세요. 출처는 각 사업 보도자료·결과보고서 링크로."

**차이**: 범위·기준·포맷·출처까지 지정하면 서포터가 바로 작업 가능. 방향 없으면 서포터도 추측하다 결국 재작업.

**대응**: 서포터 투입 전 10분 쓰더라도 **범위·기준·포맷·출처** 4가지를 명시한 의뢰서 작성. 중간 점검을 첫 1/3 시점에 한 번.

## 함정 6. 성과 측정과 리포팅 구조 누락

**원칙**: 모든 사업 유형에서 **"무엇을 어떻게 측정해 어떻게 보고할지"** 가 제안서 섹션으로 드러나야 한다. 측정·리포팅이 약하면 평가·재계약·차기 사업 연결이 모두 끊긴다.

### 왜 측정·리포팅은 사업 유형을 가리지 않는가

발주처마다 성과 리포트의 **수신처** 가 다릅니다. 수신처가 다를 뿐, 리포트 자체는 공통 요구입니다.

| 발주처 유형 | 성과 리포트의 사용처 |
|---|---|
| B2G (중앙·지자체·공공) | 정부업무평가 · 감사 · 국정감사 · 차년도 예산 편성 |
| B2B (대기업·중견) | 지속가능경영보고서 · ESG 리포트 · 임원 보고 |
| 지자체 | 지방 의회 · 주민 공청회 · 지역 경제 지표 |
| 재단 · 협회 | 이사회 · 기부자 · 후원 기업 정기 리포트 |
| 해외 사업 | 해외 파트너 · 바이어 · 본사 영문 KPI |

### 사업 유형별 측정 대상 예시

| 사업 유형 | 주 측정 대상 |
|---|---|
| 창업 교육 | 사전·사후 역량 진단 · 창업률 · 매출 발생률 · 후속 지원 연결률 |
| 소상공인 지원 | 매장 매출 변화 · 객단가 · 재방문율 · 온라인 전환 |
| 로컬 상권 | 빈 점포 입점률 · 상권 방문객 · 상인 협의체 활성도 |
| 공모전 | 수상작 유통 입점률 · 판매 실적 · 수출 건수 |
| 문화·장인 | 공동 창작물 수 · 국제 교류 지속률 · 파트너 국가 수 |
| 매칭·프로보노 | 매칭 만족도 · 수혜 조직 KPI 변화 · 재참여율 |
| 글로벌 | 해외 법인 · 해외 매출 · 해외 투자 유치 · 바이어 계약 |

### 대응 구조

- 모든 제안서에 **"성과 측정 및 리포팅"** 섹션 1~2페이지 배치
- **지표 · 측정 도구 · 측정 시점 · 리포트 주기** 4열 표
- 자체 진단 도구 (ACT-PRENEURSHIP · DOGS · 6 Dimension 등) 가 적용 가능한 사업에서는 정량화 근거로 활용
- 중간·최종 리포트 외 3~6개월 후 **follow-up 리포트** 를 추가 제안

### 체크포인트

- [ ] 측정 대상이 사업 유형에 맞게 설정되었는가
- [ ] 측정 시점 (사전 · 중간 · 사후 · 추적) 이 명시되었는가
- [ ] 리포트의 **수신자 · 포맷 · 주기** 3가지가 구체화되었는가
- [ ] 후속 연결 (재계약 · 차기 사업 · 감사) 에 쓸 수 있는 포맷인가

## 함정 7. Section V (RFP 범위 밖 추가 제안) 누락

RFP 가 요구한 과업만 채우면 "적정한 2등" 제안서가 됩니다. RFP 에 없는 추가 제안 3~4건을 마지막에 담는 **Section V 보너스** 가 수주 제안서의 공통 패턴입니다.

### Section V 단골 항목

- **글로벌 연계**: 일본·인도 법인 활용, 해외 박람회 참여 지원, 글로벌 바이어 매칭
- **임팩트 리포트**: 자회사 한국사회가치평가 연계, ESG 가치 측정 방법론 적용
- **후속 투자 연계**: 라이콘(LICORN) 투자 심사, 중기부 LIPS 융자 매칭 (최대 5배)
- **사후 교류 커뮤니티**: 알럼나이 네트워크, 동료 멘토링 연결
- **추가 콘텐츠**: 영상 아카이빙, 외부 협업 전시, 공모 연계

각 항목은 **"우리만 할 수 있는 것" + "RFP 목적에 직접 기여하는 것"** 이 교집합이어야 의미 있습니다. 무관한 내용을 Section V 에 넣으면 "이 회사 과업 이해도가 부족" 으로 역효과.

**대응**: 초안 끝나기 전 "RFP 는 요구하지 않았지만 우리가 줄 수 있는 것 3~4건" 을 팀 브레인스토밍. 각 항목이 RFP 의 핵심 목적과 어떻게 연결되는지 1문장 설명 추가.
