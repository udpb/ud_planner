# 부록 A — 제출 직전 체크리스트

> 이 체크리스트는 제안서 제출 하루 전·당일에 돌립니다. 초안 완성 후에도 한 번 돌리면 좋습니다.

## 사용법

- [ ] 모든 항목을 체크박스로 만들어 프린트하거나 노션에 복사
- 각 섹션을 **순서대로** 확인 (구조 → 콘텐츠 → 형식)
- 한 항목이라도 미흡하면 팀에 즉시 공유
- 제출 후에는 이 체크리스트를 사례집에 보관 (다음 사업 재활용)

---

## 1. 구조 점검

### Value Chain 일관성

- [ ] Input → Activity → Output → Outcome → Impact 5계층이 명시적으로 보이는가
- [ ] 계층 사이에 "그래서?" 를 던졌을 때 답이 나오는가
- [ ] Output 과 Outcome 이 혼용되지 않았는가
- [ ] Outcome 수치가 근거 (사례·진단 데이터) 로 뒷받침되는가
- [ ] Impact 가 우리 사업의 현실적 기여 범위 안에 있는가

### 평가 배점 공략

- [ ] RFP 평가 항목별 배점 표를 갖고 있는가
- [ ] 내 제안서 페이지 수 분포가 배점과 대략 비례하는가
- [ ] 최고 배점 항목에 가장 두꺼운 섹션이 있는가
- [ ] 낮은 배점 항목에 과도한 페이지를 쓰지 않았는가
- [ ] 가점·감점·필수 제출 조항을 놓치지 않았는가

### Activity / Outcome 인과

- [ ] 주요 Activity 마다 기대 Outcome 1개 이상이 붙어 있는가
- [ ] 같은 Activity 가 여러 섹션에서 인용될 때 일관되게 설명되는가
- [ ] 커리큘럼 회차별로 산출물·도구·시간 배분이 명시되어 있는가
- [ ] 이론 3회 연속 구간 없이 Action Week 가 삽입되어 있는가 (3회 초과 시 AW 필수)

---

## 2. 콘텐츠 점검

### 정량 수치 구체성

- [ ] `많은` · `다양한` · `충분한` · `풍부한` · `최적의` 단어를 Ctrl+F 로 검색했는가
- [ ] 검색된 모든 모호 표현을 **숫자** 또는 **구체 명사** 로 교체했는가
- [ ] 주장 1개당 근거 (수치·사례·도구·체계) 1개 이상 붙어 있는가
- [ ] 정량 포화 분포가 한 섹션에 몰리지 않고 전체에 퍼져 있는가

### 자체 도구 명칭 정확성

- [ ] **ACT-PRENEURSHIP** (하이픈·대소문자 정확)
- [ ] **DOGS Team Building** (DOGS 대문자)
- [ ] **IMPACT 창업방법론** (IMPACT 대문자 + "6단계 18모듈 54문항" 정식 표기)
- [ ] **6 Dimension Startup Growth Model**
- [ ] **EduBot** (대문자 E, B)
- [ ] **언더베이스 LMS**
- [ ] **라이콘(LICORN)** · 라이콘 = Lifestyle & Local Innovation uniCORN
- [ ] **아시아투모로우** (플랫폼, 2024 런칭)
- [ ] 자체 도구명이 제안서 내에서 변형·약어·소문자화 되지 않았는가

### 4중 지원 체계 표기

- [ ] 코치·멘토를 **단일 명칭** 으로 쓰지 않았는가 ("단일 코치" 금지)
- [ ] 전문 멘토단 + 컨설턴트 풀 + 전담 코치 (액션코치) + 동료 네트워크 4레이어로 표기했는가
- [ ] AI 코치 (EduBot) 가 **별도 레이어·별도 상품** 처럼 표기되지 않았는가 (AI 는 강점 언급만)

### Section V 보너스

- [ ] RFP 범위 밖 추가 제안 3~4건이 있는가
- [ ] 각 추가 제안이 **제안사 고유 자산 × RFP 목적 기여** 의 교집합인가
- [ ] 글로벌 / 임팩트 리포트 / 후속 투자 / 사후 커뮤니티 / 아카이빙 중 1~2건 포함되어 있는가
- [ ] Section V 항목마다 **실현 가능한 구체 경로** (구체 박람회 이름·플랫폼·파트너명) 가 붙어 있는가

### 브랜드 메시지

- [ ] 법인명이 "언더독스" 로 통일되어 있는가 (유디임팩트는 법적 섹션에만)
- [ ] "약자·취약층·동정" 프레임 없이 Underdog 이 재정의된 맥락으로 쓰이는가
- [ ] 미션·비전·태그라인이 필요한 위치에 원문 보존 인용되어 있는가
- [ ] Core Values (협업·혁신·성과·책임) 가 조직/팀 소개 섹션에 배치되어 있는가

## 3. 형식 점검

### 오타·비문

- [ ] 제안사명·발주처명·담당자 이름 오타 없음
- [ ] 사업명·프로그램명·수치 오타 없음
- [ ] 국문/영문 병기 일관성 (예: 액트프러너(Act-Preneur), K-souvenir)
- [ ] 영문·숫자·기호 앞뒤 공백 규칙 일관성
- [ ] 반복 표현 (예: "진행함" · "운영함") 적절한지 최종 검토

### 파일·디자인 정렬

- [ ] 파일명이 RFP 규칙을 따르는가
- [ ] 파일 형식 (PDF · 한글 · PPT) 이 RFP 가 요구한 형식인가
- [ ] 파일 크기가 제한 이내인가
- [ ] 페이지 수가 상한 이내인가 (본문·부록 포함 여부 확인)
- [ ] 폰트 통일·폰트 크기 일관성
- [ ] 이미지·표·그래프 정렬
- [ ] 페이지 번호·헤더·푸터 일관성
- [ ] 링크·목차 내비게이션 정상 동작

### 연도 수치

- [ ] 2025년·2024년 등 연도 표기 일관성 (내년·올해·작년 등 상대 표현 혼용 주의)
- [ ] 누적 실적 수치가 최신값인지 (`src/lib/ud-brand.ts` 기준. [부록 B](./b-ud-assets.md))
- [ ] 과거 사례의 연도·금액·참가자 수가 정확한지

### 제출 정보

- [ ] 제출 기한 · 제출 방법 (온라인 / 방문 / 우편) 최종 확인
- [ ] 필수 제출 서류 (사업자등록증 · 재무제표 · 이행보증 · 인감 등) 준비 완료
- [ ] 제출 담당자 · 연락처 기재 정확
- [ ] 백업 파일 다른 드라이브·이메일에 보관

## 4. 발주처 타입별 추가 점검 (Ch.10 참조)

### B2G 일 때

- [ ] 지나치게 혁신적·실험적 표현이 없는가
- [ ] 매출·ROI·수익 극대화 같은 B2B 용어가 섞이지 않았는가
- [ ] 안전·보고·개인정보·리스크 섹션이 한 페이지 이상인가
- [ ] 성과 지표가 정부업무평가·감사에 쓸 수 있는 포맷인가

### B2B 일 때

- [ ] 정부업무 용어 (추진 체계·자생력 등) 가 섞이지 않았는가
- [ ] 매출·전환률·재참여율 같은 비즈니스 KPI 가 중심에 있는가
- [ ] 지속가능경영보고서·ESG 리포트 포맷 제공 약속이 있는가
- [ ] 재계약 구조 (내년·내후년 확장 경로) 가 암시되어 있는가

### 재계약 일 때

- [ ] "처음 뵙는" 어조가 모두 제거되었는가
- [ ] 지난해 성과 요약 + 올해 개선안이 앞쪽에 배치되어 있는가
- [ ] 작년 대비 KPI 가 업그레이드되었는가
- [ ] 작년 아쉬움 포인트에 대한 구체적 개선안이 있는가
- [ ] 작년 제안서를 그대로 복붙한 섹션이 없는가

---

## 5. 팀 합의·최종 승인

- [ ] PM · 본부장 · 대표 검토 완료
- [ ] 재무·회계·법무 관련 사항 확인 (예산 집행·이행 보증·계약 조항)
- [ ] 외부 전문가 (필요시) 피드백 반영
- [ ] 제출 전 **최소 24시간** 의 냉각기 확보 (밤샘 제출은 오타·비문 급증)

## 최종 한 문장 점검

제안서 전체를 덮고, 다음 한 문장을 스스로에게 던집니다.

> **"이 회사를 모르는 평가위원이 이 제안서만 보고 (1) 우리가 RFP 를 정확히 읽었고 (2) 근거를 갖고 답하며 (3) 실행할 능력이 있다는 확신을 갖는가?"**

세 가지 모두 "그렇다" 면 제출. 하나라도 주저된다면 해당 부분 재검토.
