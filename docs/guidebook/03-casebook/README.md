# Part 3 — 케이스북

> 이 페이지를 읽으면 각 사례를 어떤 포맷으로 읽어야 하고, 무엇을 뽑아낼지 알게 됩니다.

## 이 케이스북이 가장 중요한 이유

책의 허리입니다. 신입 PM 이 감을 빠르게 잡으려면 **좋은 사례를 많이, 구조적으로** 읽는 게 가장 효율적입니다. 추상적인 원칙은 Part 1~2 에 있지만, 실제로 어떻게 적용되는지는 여기서 보입니다.

케이스는 계속 늘어납니다. 수주든 탈락이든 새 사업마다 한 건씩 추가되도록 포맷을 고정했습니다.

## 동일 포맷 — 6개 섹션

모든 케이스가 같은 목차를 따릅니다. 다른 케이스로 넘어가도 같은 위치에서 같은 정보가 나옵니다.

1. **사업 한 줄 요약** — 발주처·금액·기간·대상·핵심 과업
2. **핵심 메시지** — 원문 보존 (제안사가 직접 쓴 슬로건·선언)
3. **Impact Value Chain 5계층** — Input → Activity → Output → Outcome → Impact
4. **차별화 포인트 3개** — 다른 경쟁사와 구분되는 고유 요소
5. **언더독스 자체 도구·네트워크가 어떻게 녹았나** — ACT-PRENEURSHIP·DOGS·IMPACT·라이콘 등이 이 사업에서 작동한 방식
6. **신입 PM 이 이 케이스에서 배울 점** *(가장 중요)* — 재사용 가능한 구성 요소·사고법·주의사항

## 케이스 읽을 때 유의점

- **원문 메시지 보존**: 섹션 2의 슬로건·선언은 제안사가 실제로 쓴 표현입니다. 다른 사업에 복붙하면 맥락이 사라져 어색합니다. **감을 잡기 위한 참고**, 재사용 시에는 새 사업 맥락으로 재창작.
- **5계층은 참고 구조**: 실제 제안서가 꼭 이 5계층 순서로 쓰여 있진 않습니다. 가이드북이 사후에 구조를 정리한 것. 원문 제안서에서는 평가 배점 순서로 배치됩니다.
- **차별화는 고유 요소**: "우리 회사가 잘한다" 가 아니라 "이 사업의 이 과업에서, 이 회사만 이렇게 접근한다" 는 의미. 다른 사업에 그대로 가져다 쓸 수 없습니다.
- **언더독스 자산 확인**: 수치·도구명은 [부록 B 언더독스 자산 참고표](../appendix/b-ud-assets.md) 의 현재값을 기준으로 검증하면서 읽으세요. 케이스 원본에 쓰인 숫자는 당시 시점 수치입니다.
- **섹션 6에 제일 오래 머물기**: "이 케이스에서 배울 점" 이 가이드북 케이스북의 본질입니다. 원본 요약보다 이 섹션에서 얻을 게 많습니다.
