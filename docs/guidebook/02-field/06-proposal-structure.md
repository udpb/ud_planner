# 제안서 장표 구성법

> 이 챕터를 읽으면 RFP 를 받고 "어떤 목차로, 몇 페이지씩, 어떤 톤으로" 장표를 짤지 기준을 갖게 됩니다.

## 1. 표준 섹션 구조 (I~V)

언더독스 제안서의 기본 골격은 **5개 섹션**. RFP 가 목차를 지정하면 반드시 따르고, 지정이 없으면 이 구조를 기본으로 합니다.

| 섹션 | 제목 | 핵심 | 전체 대비 분량 |
|------|------|------|-------------|
| **I** | 일반 현황 | 회사 소개 · 조직 · 실적 · 재무 | 10~15% |
| **II** | 기본 계획 | 제안 배경 · 전략 · 인력 · 로드맵 · 기대 성과 | 20~25% |
| **III** | 수행 계획 | **제안서의 심장** — 커리큘럼 · 코치 · 임팩트 · 운영 상세 | 40~50% |
| **IV** | 사업 관리 | 품질 보증 · 보고 체계 · 예산 | 10~15% |
| **V** | 기타 추가 제안 | RFP 범위 밖 보너스 3~4건 — **핵심 차별화 지점** | 5~10% |

### Section III 가 절반인 이유

평가위원은 "정말 할 수 있는가" 를 여기서 판단합니다. 구체적인 시간표 · 세션별 내용 · 코치 프로필 · Action Week 설계 · 측정 계획이 여기 들어갑니다. **배점 최고 항목이 대개 여기에 대응됩니다.**

### Section V 를 빼면 안 되는 이유

RFP 가 요구하지 않은 것을 "우리는 이것도 할 수 있습니다" 라고 제안하는 곳. 여기가 경쟁사와 가장 차이 나는 영역입니다. 예시:
- "글로벌 연계 프로그램 (도쿄 지사 활용)"
- "사후 임팩트 리포트 (SROI 측정)"
- "알럼나이 네트워크 연결"
- "자체 투자사(라이콘) 연계 가능성"

## 2. One-Page-One-Thesis 원칙

**한 페이지 = 한 가지 주장.**

```
[컬러 헤더 바] → 섹션 구분
[서브 헤드] → 맥락 설정
[ONE BOLD 선언적 헤드라인] → 이 페이지가 말하는 한 가지
[구조화된 시각 자료] → 헤드라인을 뒷받침하는 증거
```

### 왜 이렇게 해야 하나

평가위원은 **제안서를 처음부터 끝까지 정독하지 않습니다.** 페이지를 넘기며 헤드라인만 훑고, 관심 있는 곳만 깊이 읽습니다. 헤드라인이 없거나 한 페이지에 여러 주제가 섞이면 "정리 안 된 제안서" 인상을 줍니다.

### 좋은 헤드라인 vs 나쁜 헤드라인

| 나쁜 예 | 문제 | 좋은 예 |
|---------|------|---------|
| "커리큘럼 구성" | 무슨 주장인지 모름 | "실행 중심 12주 커리큘럼 — Action Week 3회로 MVP 산출률 90% 달성" |
| "조직 구성" | 나열형 제목 | "전담 PM 배치 + 4중 지원 체계로 1,500명 동시 운영 가능" |
| "예산 계획" | 예산표만 있으면 안 됨 | "프로그램 투자 비율 91.2% — 성과에 집중하는 예산 구조" |

## 3. 정량 포화 원칙

**"많은", "다양한", "상당한" 같은 모호한 표현을 금지합니다.**

```
❌ "많은 코치를 보유하고 있습니다"
⭕ "전국 800명의 전문 코치 풀을 보유하고 있습니다"

❌ "풍부한 경험을 가지고 있습니다"
⭕ "10년간 498개 프로그램, 20,211명의 창업가를 육성했습니다"

❌ "다양한 파트너와 협력합니다"
⭕ "520개 이상의 글로벌 파트너, 96개 지역에서 동시 운영합니다"
```

숫자는 제안서 전체에 **5~10개 분포** 시킵니다. 한 섹션에 몰아넣으면 나머지가 빈약해 보입니다.

## 4. 조직 구조 표현법

**"전담" 역할을 반복 강조합니다.** 평가위원은 "이 사업에 전담인가, 겸임인가" 를 봅니다.

```
대표 (총괄 책임)
└ 사업 PM (본부장급, 전담)
  └ 기능별 리더 (기획/운영/육성콘텐츠/홍보)
    └ CM (Coach Manager) — 코치 운영 총괄
      └ 전국 코치진 + 지역 PM
```

가능하면 실제 이름 · 사진을 사용합니다. "배정 예정" 보다 "홍길동 본부장 (전담)" 이 평가에서 확실히 우위입니다.

## 5. 4중 지원 체계 표현법

단일 코치를 표현하지 마세요. 반드시 **레이어 구조로 도식화**:

1. **전문 멘토단** — 분야별 전문가 300명+ 풀, 분기별 전문 조언
2. **컨설턴트 풀** — 심화 분야 1:1 컨설팅 (회계, 법률, 투자, 글로벌)
3. **전담 코치 (액션코치)** — 주간 1:1 코칭, 실행 견인, 진행 점검
4. **동료 네트워크** — 코호트 러닝, 알럼나이 커뮤니티, 협업 연결

이 구조는 **모든 제안서에** 들어갑니다. 사업 특성에 따라 각 레이어의 구체 인원과 역할만 조정합니다.

## 6. 예산 표현법

예산은 제안서의 주인공이 아닙니다. **마지막에 담백하게** 표현합니다.

핵심 메시지:
- "프로그램 투자 비율 91.2%" 처럼 **효율성을 프레이밍**
- 직접비 비율을 높게 보여주기 (B2G 는 70% 이상 권장)
- 마진은 보수적으로 (10~15%), 과도하면 감점

**절대 하면 안 되는 것:** 예산을 먼저 잡고 커리큘럼을 거기 맞추기. 평가 배점 최고 항목이 "커리큘럼 30점" 인데 예산에 맞춰 세션을 줄이면 30점을 날리는 것입니다.

## 7. 제안서 작성 순서 (장표 작성 전에 할 것)

이건 Ch.5 (함정) 1위와 같은 이야기입니다:

```
1. Impact Value Chain 먼저 (Ch.4)
2. 평가배점 분석 (Ch.3 RFP 첫 24시간)
3. 핵심 메시지 3개 확정
4. 목차 + 페이지 배분
5. 그다음에야 장표 작성
```

**코오롱 프로보노에서 배운 교훈:** Value Chain 없이 장표부터 쓰면 2주 무한수정. Value Chain 잡은 뒤 1.5일 만에 완성.

## 8. 스스로 점검 3문항

- [ ] 각 페이지에 **한 가지 선언적 헤드라인** 이 있는가?
- [ ] 모호한 수량 표현("많은", "다양한") 이 **0개** 인가?
- [ ] Section V (추가 제안) 에 **3건 이상** 차별화 항목이 있는가?
