# Ch.4 Impact Value Chain 사고법

> 이 챕터를 읽으면 Input → Activity → Output → Outcome → Impact 5계층을 구분하고, "그래서?" 테스트로 인과 사슬을 점검할 수 있게 됩니다.

## 왜 Value Chain 인가

제안서에서 가장 흔한 실패는 "우리가 할 활동" 과 "결과로 만들어질 임팩트" 사이에 **인과 사슬이 끊겨** 있는 것입니다. 예를 들어:

> 창업가 30명 대상 8주 교육 → 지역 경제 활성화

"교육 30명 → 경제 활성화" 사이에 어떤 단계로 어떻게 이어지는지가 없습니다. 평가위원도, 발주처도, 심지어 우리 자신도 설득할 수 없습니다. 이 끊김은 "대충 쓰면 들킨다" 는 수준이 아니라, **수주의 근본 설득력을 무너뜨리는 구조 결함** 입니다.

Impact Value Chain 은 이 끊김을 방지하는 가장 단순한 도구입니다. 활동에서 임팩트까지 5계층으로 나눠, 계층 사이마다 "그래서?" 를 던져 인과가 말이 되는지 봅니다.

## 5계층 정의

| 계층 | 정의 | 예시 |
|------|------|------|
| **Input** | 투입 자원 — 돈·사람·시간·공간·네트워크 | 예산 5억, PM 2명, 코치 10명, 강의실 5개, 지역 협력 기관 3곳 |
| **Activity** | 수행 활동 — 우리가 실제로 한 일 | 8주 창업 교육 진행, 코칭 40회, Demo Day 1회 |
| **Output** | 즉각적 산출물 — 활동의 직접 결과 (수량화 가능) | 수료생 28명 배출, 사업계획서 28건 도출, MVP 제품 15개 제작 |
| **Outcome** | 변화된 상태 — 참여자/현장에 일어난 중단기 변화 | (사업 유형별) 창업: 창업률 40%·월매출 평균 200만원 / 소상공인: 매장 매출 30% 증가·객단가 20% 상승 / 로컬상권: 빈 점포 입점률 15% 감소·재방문율 40% / 공모전: 수상작 유통 입점 3건·수출 2건 / 문화장인: 국제 공동 전시 2건·파트너 국가 교류 정례화 |
| **Impact** | 장기 사회 가치 — 사업 종료 이후 지속되는 시스템·생태계 변화 | 지역 청년 유출 저하, 골목 상권 매출 20% 증가, 자생 상인회 1개 결성 |

핵심 구분 두 가지만 체득하면 됩니다.

### Output vs Outcome

가장 많이 혼동하는 경계입니다.

- **Output** = "뭐가 나왔나" (행사가 끝난 직후 셀 수 있음)
- **Outcome** = "뭐가 달라졌나" (참여자·현장에 실제 변화)

예 — 사업 유형별:

| 상황 (사업 유형) | Output | Outcome |
|---|---|---|
| 창업 교육 8주 | 수료생 30명, 사업계획서 30건, MVP 15개 | 창업 준비도 70% 상승, 실제 창업 12팀, 월매출 평균 200만원 |
| 소상공인 성장 지원 6개월 | 리뉴얼 완료 매장 25개, 운영 교육 수료 25명 | 평균 매출 30% 증가, 객단가 20% 상승, 재방문율 40% |
| 로컬 상권 브랜드 위크 | 방문객 10,000명, 참여 상점 40개, 팝업 운영 5회 | 상권 매출 30% 증가, 빈 점포 입점 3건, 재방문율 40% |
| 공모전 (디자인·기념품) | 출품작 600점, 수상작 30점 | 수상작 유통 입점 5건, 누적 매출 2억, 수출 2건 |
| 문화·장인 교류 | 공동 창작 작품 15점, 국외 교류 행사 3회 | 국제 공동 전시 2건, 참여 장인 해외 주문 5건, 교류 플랫폼 구축 |
| 소셜 임팩트·매칭 | 매칭 페어 30쌍, 공동 프로젝트 30건 | 수혜 조직 KPI 평균 2.3배 상승, 임직원 재참여 의사 80% |
| 글로벌 진출 | 현지 방문 3회, 바이어 미팅 40건 | 해외 법인 설립 3팀, 해외 매출 발생 5팀, 해외 투자 유치 2팀 |

Output 만 제시하면 "수행은 했다" 에 그칩니다. Outcome 까지 제시하면 "수행이 효과적이었다" 가 됩니다. 수주 제안서는 **Output + Outcome 두 줄 쌍으로** 써야 힘이 생깁니다.

### Activity vs Output

이것도 혼동됩니다.

- **Activity** = 동사 ("~한다", "~를 진행한다")
- **Output** = 명사·수량 ("수료생 30명", "사업계획서 30건")

"8주간 코칭을 진행한다" 는 Activity. "코칭 총 40회 × 참여자 30명 = 1,200회 1:1 코칭" 은 Output.

## "그래서?" 테스트

계층 사이 인과가 끊겼는지 확인하는 가장 쉬운 방법은 **"그래서?" 를 소리 내어 던지는 것**입니다. 예:

> Activity: 8주 창업 교육을 진행한다
> **그래서?**
> Output: 수료생 30명, MVP 15개
> **그래서?**
> Outcome: 창업 12팀, 월 매출 평균 200만원 달성
> **그래서?**
> Impact: 지역 내 창업 문화 확산, 후속 기수 모집 시 자발적 지원자 2배 증가

각 "그래서?" 에 답이 나오면 인과가 붙은 것입니다. 답이 안 나오면 그 사이에 계층이 빠진 것. 대부분의 끊김은 **Output 과 Outcome 사이** 에 생깁니다. "수료생 30명" 까지는 쉽게 쓰지만, "그래서 참여자에게 무엇이 달라졌는가" 는 자료가 필요합니다.

### 끊김의 전형적 패턴

1. **Activity → Impact 점프**: "교육 → 지역 활성화" 처럼 사이 계층을 건너뛰는 경우. 가장 흔함.
2. **Output = Outcome 혼동**: 수료생 수를 Outcome 이라 쓰는 경우. 수료는 참여의 결과이지 변화가 아님.
3. **Outcome 이 추상적**: "창업 마인드 향상" 같은 측정 불가 표현. **사업 유형에 맞는 수치** 로 교체:
   - 창업 교육: "사전·사후 ACT-PRENEURSHIP 진단 35 → 52"
   - 소상공인: "매장 평균 월매출 200만 → 260만 (30% 상승)"
   - 로컬 상권: "빈 점포 12개 → 9개 (25% 감소), 상권 재방문율 35%"
   - 공모전: "수상작 중 유통 입점 3/10팀, 누적 매출 1.5억"
   - 문화·장인: "국제 공동 창작 5건, 파트너 국가 연 3회 교류 정례화"
   - 매칭·프로보노: "수혜 조직 KPI 평균 2.3배 상승, 임직원 재참여 의사 80%"
4. **Impact 가 남의 일**: 우리 사업이 직접 기여했다고 말하기 어려운 큰 지표. "국가 GDP 상승" 같은 건 Impact 아님.

## 어떻게 현업에 적용하나 (추상 예시)

추상적으로 한 번만 연습해 봅시다. 구체적 사례는 Part 3 케이스북에 풍부합니다.

가상의 RFP: "경기도 중소 상공인 브랜딩 지원 사업, 예산 3억, 대상 20팀, 기간 6개월."

5계층 초안:

- **Input**: 예산 3억, PM 1명, 전담 코치 5명, 브랜딩 전문가 자문단 10명, LMS 1개
- **Activity**: 사전 진단 (ACT-PRENEURSHIP) → 6주 브랜딩 워크숍 → 2주 Action Week (현장 적용) → 1:1 코칭 120회 → 통합 쇼케이스
- **Output**: 수료 18팀, 브랜드 가이드 18건, 시제품 18종, 쇼케이스 방문객 2,000명
- **Outcome**: 참여팀의 평균 월 매출 150만원 → 240만원 (60% 상승), 브랜드 인지도 SNS 팔로워 평균 3배 증가, 후속 투자·대출 유치 3팀
- **Impact**: 경기도 권역 소상공인 브랜딩 사례집 1권 발간 → 타 지자체 벤치마킹 문의, 권역 상공회의소 자생적 브랜딩 모임 발족

"그래서?" 테스트를 돌려 보면 대부분 붙습니다. Outcome 단계의 수치가 근거 없이 쓰이면 약점입니다 — 과거 수행 사례에서 실측 데이터로 뒷받침하거나, "제안사 유사 사업 평균치 기준" 같이 출처를 밝힙니다.

## 사고법을 왜 굳이 배우는가

첫째, **설득력**. Value Chain 이 선명하면 평가위원이 각 계층별로 배점 항목에 매칭하기 쉬워집니다. 배점 항목이 "기대효과" 라면 Outcome + Impact 를 몰아서 강화하고, "실행 계획" 이라면 Activity + Output 을 조밀하게 씁니다.

둘째, **팀 합의**. 신입 PM 이 혼자 쓰면 대개 Activity 만 빽빽해집니다. 본부장·CM·재무 담당이 한 페이지 Value Chain 을 같이 보면서 "Outcome 이 약한데 어떤 데이터를 더 넣을까" 의사결정이 가능해집니다.

셋째, **수정 비용 절감**. Ch.5 함정 1번에서 코오롱 프로보노 사례로 다룹니다만, Value Chain 없이 장표부터 쓰면 피드백 때마다 뒤집어집니다. Value Chain 확정 후 작성하면 섹션 단위 수정만으로 완성됩니다.

## 점검 체크리스트

초안 완성 후 아래를 돌려 봅니다.

- [ ] 5계층이 각각 한 줄씩 명시되어 있는가?
- [ ] Activity → Output → Outcome → Impact 사이에 "그래서?" 를 넣었을 때 답이 나오는가?
- [ ] Output 과 Outcome 이 혼용되지 않았는가?
- [ ] Outcome 이 수치·진단 점수·증거로 뒷받침되는가?
- [ ] Impact 가 우리 사업의 현실적 기여 범위 안에 있는가?
- [ ] 배점 항목과 각 계층이 매핑되는가?
