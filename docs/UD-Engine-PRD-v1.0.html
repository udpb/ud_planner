00 · Summary

한 페이지 요약

무엇을 만드는가 / 왜 지금 이 방향인가 / 어떻게 압도적으로 좋아지는가.

북극성 — RFP 한 부를 넣으면 30~45분 안에 발주처에 제출 가능한, 당선 가능성이 측정된, 여러 과업이 통합된 기획 1차본이 나온다. 그리고 우리가 만든 모든 제안서·자산이 엔진에 영구 축적되어, 다음 제안서는 직전보다 반드시 더 좋다.

핵심 통찰 — 3기둥

① 압도적 품질

단일 생성 엔진이 평가표(rubric)에 정렬된 당선 언어를 조립. "good enough"는 시장 평균 — 우리는 증명된 win-theme + 평가위원 채점 정렬로 간다.

② 지속 학습·주입

자산·당선 제안서를 그래프로 축적, 드롭 한 번으로 자동 분류·임베딩·검수. 사람 수정·당락 결과가 학습 신호로 되먹임.

③ 멀티 과업 구조

제안서는 교육+행사운영+장소+특강연사… 여러 과업의 통합체. 과업(Workstream)을 1급 레이어로 — 시장 누구도 안 하는 wedge.

인사이트 — 벤치마크 결론

시장 선두 Responsive(관리 $600B)조차 (a) 닫힌 win/loss 학습 루프와 (b) 멀티 과업·임팩트 grounding은 못 한다. 둘 다 우리가 이미 아키텍처적으로 앞선 지점(PipelineContext·Impact Value Chain·Content Hub·Express 자동 진단). 이 두 개가 결정적 차별화다.

가장 시급한 진실

현재 production Express는 약한 turn(슬롯필링) 경로 + 모델은 실측 flash로 동작 — 품질 작업·평가(패널 83점)가 사용자에게 닿지 않는다. "압도적 품질"의 1번 과제는 단일 엔진을 production에 배선하고 본문 생성을 frontier 모델로 올리는 것.

01 · Essence

본질 — 생성기가 아니라 씽크탱크

청중: 경영진·기획. "왜 단순 AI 글쓰기 도구가 아닌가."

대부분의 제안서 AI는 생성기다 — 요청을 받아 그럴듯한 초안을 뱉는다. 우리가 만드는 건 씽크탱크(엔진)다 — 회사의 모든 기획 지능을 축적하고, 그 위에서 추론하며, 결과(당락)로 스스로 강화된다. 차이는 복리(compounding)다. 생성기는 매번 0에서 시작하지만, 엔진은 직전 제안서까지의 모든 학습을 딛고 시작한다.

축단순 생성기 (시장 평균)UD 기획 엔진 (목표)
지식flat 벡터 더미에서 검색그래프 제약 코퍼스 — Value Chain 백본, 전역 추론
출력섹션별 그럴듯한 산문평가표 정렬 — compliance matrix + 증명된 win-theme + SROI
학습없음 (매번 0)수정 diff + 당락 + 평가자 튜닝 3 플라이휠

인사이트

Anthropic 멀티에이전트 연구·Glean·Microsoft GraphRAG의 공통 교훈 — "더미를 그래프로 바꾸는 것"이 생성기를 씽크탱크로 만든다. 우리는 이미 Impact Value Chain(Impact→Input→Output→Activity→Outcome)이라는 의미 그래프를 갖고 있다. 검색 백본으로 승격하면 "이 발주처/사업유형에서 무엇이 당선되는가" 같은 전역 질문에 답할 수 있다(vector RAG로는 불가).

02 · Market

시장·벤치마크 — 우리의 wedge

청중: 경영진·기획. 외부 리서치 기반(출처 §9).

시장은 한 패턴으로 수렴했다. 선두 도구(Responsive/RFPIO, Loopio, AutogenAI, GovDash, Tribble, Arphie, Grantable)는 모두 동일 파이프라인 — 요구사항 분해(shredding) → compliance matrix → RAG 1차 초안 → 신뢰도·인용 → 사람 검수. 속도("5분 내 초안")는 이미 상품화되어 차별화 지점이 아니다.

도구핵심당선 차별화win/loss 루프

AutogenAI고객별 language engine · win-themewin-rate 30%↑(자체)불투명

"good enough" vs "압도적" — 무엇이 가르나. 리서치(Arphie 평가 가이드 + Shipley/APMP) 결론: 당선 출력은 더 나은 산문이 아니라 평가위원이 채점하는 방식에의 구조적 정렬이다.

  • 산문이 아니라 평가표를 채점하라 — 가중 배점에 노력 집중.
  • Compliance는 가장 예방 가능한 패배 — 누락 요구사항 하나로 채점 전 실격.
  • 구체성이 역량 주장을 이긴다 — "효율 개선" ✕ → "14일 검토를 4일로" ○. (우리의 SROI·Before/After·코치 풀이 구체성 금광.)
  • Win-theme = 차별점 + 고객 편익 + 정량 가치 + 증거. "증명 못 하면 말하지 마라."

우리의 wedge — 시장 공백 4가지

  1. 멀티 과업 제안서. 모든 벤치마크 도구는 단일 스레드 Q&A 조립용 — 교육+행사+장소+연사를 하나로 모델링하는 도구는 없다. 가장 명확한 차별화.
  2. 한국 B2G 사회공헌 framing. GovDash 등은 US 연방·영문 Shipley 전용. B2G/B2B 자동 진단 + "일반 전략 → 사회공헌사업" 재framing은 무경쟁.
  3. 임팩트·SROI grounding을 구체성 레이어로. 임팩트 인접 Grantable조차 SROI value-chain 모델링 없음.
  4. 닫힌 win/loss 학습 루프. 선두 Responsive조차 안 하는 것 — 제출 1차본 → 당락 → win-theme·자산 재가중.

03 · Architecture

제품 구조 — 4 레이어

청중: 개발/AI + 기획. "엔진이 실제로 어떻게 도는가."

[주입] RFP·당선제안서·자산 ──drop/url/paste──▶ 멱등 ingest + 자동분류·태깅·임베딩 + 검수큐 [지식] 그래프 제약 코퍼스(Value Chain 백본) · 과업유형 태깅 · 신선도/출처 · Contextual RAG + GraphRAG [입력] RFP 파싱 → 과업 N개 자동 분해 → PM은 결정 지점만 [생성] gather-병렬(리서치·코치·자산) → assemble-단일(7섹션·plan-then-write) → verify-분리(faithfulness·인용) [출력] compliance matrix + typed win-theme + 과업별 디테일 + SROI + .pptx/.md (= 측정 Rubric self-score) [플라이휠] ① 사람 수정 diff ② 당락 결과 ③ 평가자 튜닝 → 다음 제안서 더 좋아짐

지식 — 더미가 아니라 그래프. 모든 자산·코치·당선 청크·통계를 (Value Chain 단계 × RFP 섹션 × 과업유형 × 증거유형) 노드로 태깅하고 태그를 엣지로. 검색은 그래프 이웃으로 제약(Glean), 전역 질문은 GraphRAG 커뮤니티 요약(Microsoft, 전역 포괄성 +50~70%). 모든 노드에 신선도·출처 필드 — 통계·당선 주장이 가장 빨리 부패하고 평가위원이 팩트체크하는 지점.

생성 — 병렬 수집 / 단일 조립 / 분리 검증. 제안서는 강하게 상호의존적(섹션 정렬 필수)이라 전면 멀티에이전트 작성은 Anthropic이 명시한 안티패턴. 병렬 수집(리서치·코치·자산)은 독립 스레드로, 본문 7섹션은 공유 컨텍스트 단일 조립(plan-then-write, STORM +25%), 검증은 자동 진단 4종 + 인용 grounding pass로 분리.

출력 — 과업(Workstream) 구조 (ADR-019). 프로젝트 = N개 타입된 과업. 7섹션·키메시지는 과업 집합 위의 투영/합성. 각 과업은 type · 연결 RFP 배점 · 디테일 · 자산 · 증거 · 예산 슬라이스 · 뒷받침 포인트를 가진다. ProgramProfile 6 과업유형(각 RFP 배점 1:1)을 골격으로 승격.

04 · Win Science

당선 과학 → 측정 가능한 Rubric

청중: 기획 + AI. "당선"을 막연한 목표가 아니라 기계가 채점하는 점수로.

한국 공공은 협상에 의한 계약 — 기술 70 : 가격 30(60:40~80:40 조정). 가격은 정량 공식이라 깎아도 거의 못 얻는다(추진전략 70에서 승부). 정성 배점에서 추진전략(~30)이 최대 레버, 사업이해도·차별성이 가장 싼 점수 — 정확히 보일러플레이트 AI가 실패하는 곳.

엔진 self-score Rubric — 제출 전 시뮬레이션. 평가위원 시트를 그대로 최적화 대상으로.

당선 레버 — win-theme을 typed object로

산문이 아니라 객체로: {차별점, 고객편익, 정량가치, 증거[]}. 증거[]가 비면 hard validation error ("증명 못 하면 말하지 마라"). 금지어("최고 수준"·"world-class"·"풍부한 경험")는 자동 차단 → 숫자·이름·날짜로 치환. 절대 수치 조작 금지(정직한 데이터가 가짜 통계를 이긴다).
