## 0. 한 문장 정의

**RFP를 넣으면, 우리 자산과 작년 경험 위에서, PM이 날을 세워 변형할 수 있는 "프로그램 기획(커리큘럼)"이 자라난다.**
제안서를 쓰는 게 아니다. **기획 골자**를 만든다. 제안서는 그 골자의 하류 산출물일 뿐.

---

## 1. 정본 플로우 (고정 — 변하지 않음)

```
① RFP 분석
   → ② 기획의도        ★새 단계. 작년 운영·담당자 의도·암묵지를 잡는 "맥락의 못"
   → ③ 커리큘럼(프로그램)  우리 자산으로, PM이 변형
   → ④ 코치 추천         커리큘럼에 맞춰
   → ⑤ 예산 + SROI       위에서 자동 수렴 (SROI=렌즈)
   → ⑥ 전체 기획 골자
```

이 순서·관점은 사용자가 못박은 불변값이다. 바뀌는 것은 **순서가 아니라 "느낌"** — 딱딱한 관문에서 자라나는 캔버스로.

---

## 2. 왜 지금 "맥락없이 딱딱"한가 — 진단 3가지 (재설계의 출발점)

| # | 문제 | 증상 | 해법 |
|---|------|------|------|
| 진단1 | **기획의도 레이어 부재** | RFP→바로 커리큘럼. 산출물이 매달릴 '못'이 없어 근거 없이 툭 떨어짐 | ②기획의도를 **눈에 보이는 단계**로 신설 |
| 진단2 | **산출물이 완성품으로 투척** | PM이 자산을 인지·이해·변형할 손잡이 없음. AI가 주인 | 모든 AI 산출 = **초안 + 왜 + 출처 + 편집 핸들**. 결정·변형은 PM |
| 진단3 | **UI가 관문(pass/fail)** | 상단 6칸 stepper + `48/70` 점수 + 체크/엑스 게이트 = 채점 받는 느낌 | 점수·게이트 후퇴. **위→아래로 맥락이 흐르며 손대면 발전하는 캔버스** |

---

## 3. UX 핵심 원칙 4개 ("딱딱 → 디벨롭")

1. **맥락은 항상 보인다.** 상단 "맥락 띠"(기획의도 요약)가 모든 단계에 떠 있어, 각 산출물이 "왜"에 매달린 걸 PM이 늘 본다.
2. **AI는 깔고, PM은 결정한다.** AI 산출은 전부 초안(draft). PM이 끌고·바꾸고·삭제·수락. 강제값 없음(제0원칙: 모든 규칙은 default).
3. **모든 결과에 출처와 이유.** 회차·코치·예산 각 항목 옆에 `UD 자산 · ACT Canvas M3` 출처 + `← 기획의도: 체류형` 이유.
4. **대화로 발전한다.** 각 단계 하단 디벨롭 바 — "이 회차 더 실습 중심으로" 같은 자연어로 기획을 비튼다(피드백 ⑤ 챗봇형).

## 4. 데이터·지식 흐름 (단계 사이로 무엇이 흐르나)

```
RFP 문서
  │  [추출]
① ─ RFP 핵심(목표·대상·기간·예산·평가배점) + 닮은 자산 자동매칭
  │  넘기는 지식 ▸ 목표·제약·평가배점
② ─ 확정된 기획의도("왜 이렇게")          ← AI 초안 + PM 대화 보정(하이브리드)
  │  넘기는 지식 ▸ 기획의도·차별점·리스크   (★아래 모든 '왜'의 출처)
③ ─ N회차 커리큘럼(회차별 '왜'+자산출처)   ← 의도 위에서, 자산으로, PM 변형
  │  넘기는 지식 ▸ 회차 구성·필요 역량·실습 비중
④ ─ 회차별 코치 배정 + 근거                ← 715명 5축 매칭, PM 교체
  │  넘기는 지식 ▸ 코치·인건비 단가
⑤ ─ 예산표 + SROI 해석(렌즈)              ← 자동 적산 + proxy 실시간 호출
  │  넘기는 지식 ▸ 총사업비·사회가치
⑥ ─ 전체 기획 골자(섹션별 출처 추적)        ← ①~⑤ 조립 → (옵션) 임팩트리포트 핸드오프
```

**지식 누적 모델:** 단계가 내려갈수록 컨텍스트가 *쌓인다*. ②의 의도가 ③④⑤⑥ 어디서든 "왜"로 소환 가능. 이게 "맥락이 안 끊긴다"의 구현.

## 5. 화면별 세부 IA (6화면)

> 공통 골격: 한 화면 = 6섹션 세로 누적(아코디언/점진공개). 상단 얇은 띠 = 현재 위치 + 다음 1액션 + 기획의도 요약(진단2·3 흡수). 점수판 없음.

### ① RFP 분석
- **받음:** RFP 문서. **남김:** 구조화 RFP 핵심 + 자산 매칭.
- **요소:** 업로드 파일 칩 · 추출 사실 그리드(목표/대상/기간·예산/평가배점) · "닮은 우리 자산" 자동매칭 리스트(유사도) · `다음—기획의도` CTA.
- **엔진(재사용):** `ai/parse-rfp` · `asset-registry.matchAssetsToRfp` · `program-profile`.

```
┌ ① RFP 분석                        [분석 완료] ┐
│ 📄 2026_카카오제주_공고.pdf  14p              │
│ ┌목표┐┌대상┐┌기간/예산┐┌평가배점┐            │
│ │정주 ││예비 ││4개월·3억││이해30 ││            │
│ └────┘└────┘└────────┘└──────┘            │
│ ▌닮은 우리 자산 · 자동매칭                     │
│   · 2025 강원 로컬창업 (0.81)                 │
│   · ACT Canvas M1~M4 · 제주 코치 38명         │
│            여기서 뽑힌 핵심이 ②~⑥의 출발점     │
└──────────────────── 다음—기획의도 → ──────────┘
```

### ② 기획의도 ★새 단계 (맥락의 못)
- **받음:** RFP핵심 + 작년 유사사업 + 자산. **남김:** "왜 이렇게 가는가".
- **방식 = 하이브리드(사용자 채택):** AI가 의도 초안 카드(목표해석·작년대비·차별점·리스크)를 깔고, **확신 낮은 2~3장에 "?" 핀** → PM이 대화로 채움. 채워진 칸이 ③의 '왜'로 직결.
- **왜 하이브리드:** 작년·담당자 암묵지는 사람만 안다(→물어야) + 백지 인터뷰는 지침(→AI가 깔아 부담↓) + 날은 PM 의도에서 섬(→핀포인트 강제). 자동=속도·맥락, 인터뷰=암묵지·날카로움 → **트레이드오프 아니라 순서.**
- **엔진:** `program-design`(D0~D8 brain) · `ai/strategic-notes` · 작년 유사사업 = `program-profile.profileSimilarity` + WinningProposalDoc.

```
┌ ② 기획의도 — 왜 이렇게 가는가 ┐
│ ✓ 목표해석(확정): 정주=체류형 창업으로                │
│ ? 작년대비(대화): 작년 이론↑ → 이번엔? ……★PM 응답   │
│ ✓ 차별점(확정): 제주 코치 38명 밀착                  │
│ ? 리스크(대화): 중도이탈·정주실패 우려?               │
│ ─ 대화 ─ 🤖 코칭 더 키울까요?  👤 네, 액션위크 1회   │
│ ↓ 채워진 의도 → ③ 각 회차의 "왜"로 내려감            │
└──────────────────────────────────────────────┘
```

### ③ 커리큘럼 (프로그램) — 심장부
- **받음:** 기획의도. **남김:** N회차(회차별 '왜'+출처).
- **요소:** 회차 카드 = `회차번호 · 제목 · 자산출처 칩 · ← 기획의도 '왜' · [끌기/교체/편집] 핸들`. 변형 중 상태 표시. 규칙(액션위크 등)은 **제안만**(수락/무시). 하단 디벨롭 바.
- **엔진:** `program-design.generate-plan`(D0~D8·운영유형 T1~T5·흐름문법) · `curriculum-ai`(방법론) · `asset-registry`.

```
┌ ③ 커리큘럼 · 8회차      PM이 주인—끌고·바꾸고·삭제 ┐
│ [3] 제주 체류형 비즈모델  🧩UD자산·ACT M3  ⋮⟳✎    │
│     ↳ 왜—작년 "체류형 강화" 의도에 대응             │
│ [4] 로컬코치 1:1 실습   🔁변형중 이론→실습  ⋮⟳✎    │
│     PM "코칭↑" 의도대로. AI 대체 자산 2개 제안       │
│ [5] 중간점검·액션위크   💡규칙 제안(강제 아님)  ⋮✎   │
│ 💬 "이 회차 더 실습 중심으로" — 대화로 디벨롭  [↗]   │
└──────────────────────────────────────────────┘
```

### ④ 코치 추천
- **받음:** 회차별 필요 역량. **남김:** 회차별 코치 배정 + 근거.
- **요소:** 코치 카드 = `이름 · 담당 회차 · 매칭 근거(지역/이력/협업) · 적합도 점수 · [다른 후보/교체]`. 5축: 키워드40·과업30·지역15·등급10·이력5.
- **엔진:** `coaches.coach-recommender` + coach-finder/Supabase `coaches_directory`(715명). 단가는 ⑤로 전달.

### ⑤ 예산 + SROI
- **받음:** 커리큘럼 + 코치. **남김:** 예산표 + SROI 해석.
- **요소:** 예산 자동 적산표(강사·운영·자산·기획 → 총사업비·마진) · **SROI 블록 = `1:4.2 [렌즈·최대화 아님]` + 정상범위 1:1~1:10 + 가정 보기 + 민감도**.
- **★SROI 불변 원칙:** 높을수록 좋은 게 아니다. 비율 최대화·랭킹 금지. "이 숫자가 어떤 가정 위에 섰나"를 비추는 도구. proxy 값은 impact-measurement **실시간 호출**(하드코딩 금지).
- **엔진:** `impact/forecast`(Wave M) · `impact/db`(proxy read) · 예산 적산. 핸드오프 = `impact/handoff.requestOfficialReport`.

### ⑥ 전체 기획 골자
- **받음:** ①~⑤ 전부. **남김:** 읽히는 기획서 + (옵션) 임팩트리포트 핸드오프.
- **요소:** 섹션 아웃라인(배경·전략·커리큘럼·코치·예산/SROI) 각 섹션에 **출처 칩**(`← ②기획의도` 등) → 맥락 추적. CTA: 내보내기 / 정밀편집(Deep) / 임팩트리포트.
- **엔진:** `ai/proposal-section` 등 조립. Deep Track(6스텝)은 정밀화 옵션으로 보존.

## 6. 현재 구현 → 재설계 매핑 (무엇을 버리고 무엇을 살리나)

| 현재 (딱딱) | 재설계 | 처리 |
|---|---|---|
| 상단 6칸 stepper | 얇은 위치·다음액션 띠 | 교체 |
| `기획 품질 48/70` 점수판 | 제거(검수는 ⑥/Deep에서) | 후퇴 |
| 체크/엑스 게이트 카드 | 규칙=제안(수락/무시) | 성격 전환 |
| RFP→바로 커리큘럼 | **②기획의도 신설** | 추가 |
| AI 완성품 투척 | 초안+왜+출처+핸들 | 컴포넌트 재설계 |
| 자산 리스트만 노출 | 회차에 박힌 출처+변형 | 재배치 |
| 경쟁 진입점 7개 | 단일 6섹션 워크스페이스 | ADR-029(BR-WS-2) |

**엔진은 0% 버린다.** 자동지능 ~20종 전부 재사용([HANDOFF](../../HANDOFF.md) "보존 자산"). 바뀌는 건 **UX/IA와 컴포넌트 껍데기**뿐.

## 7. 빌드 순서 제안 (확정 후 브리프로 분해)

- **BR-WS-3** ②기획의도 단계 신설(하이브리드 — AI 초안 카드 + "?" 핀 + 대화) — 가장 큰 갭, 먼저.
- **BR-WS-4** ③커리큘럼 카드 재설계(초안+왜+출처+드래그/교체/편집 핸들) — 심장부.
- **BR-WS-5** 공통 껍데기(맥락 띠·위치/다음액션 띠·점수판 제거·디벨롭 바) + 단계 간 지식 thread 배선.
- **BR-WS-6** ④코치·⑤예산/SROI·⑥골자를 새 껍데기에 정렬 + 출처 칩.
- **BR-WS-2** 경쟁 라우트 제거(express·v2·program-design·impact-forecast·brain) — 마지막, 검증 후.
- **임팩트 라이브:** impact-measurement `feat/service-api` 배포 + `SERVICE_API_TOKEN`(양쪽) + 로컬 `IMPACT_MEASUREMENT_DATABASE_URL`(피드백 ④).

> 메인은 직접 빌드하지 않는다(ADR-020). 위 브리프를 자급자족으로 써서 서브에 위임, 5섹션 검증 보고로 받는다.

## 8. 불변 제약 (빌드 시 위반 금지)

- SROI는 **렌즈** — 최대화/랭킹 금지.
- proxy 값 = **실시간 호출**, 하드코딩 금지.
- 규칙은 전부 **default(제0원칙)** — 강제값 금지. 모호하면 사람에게 위임(턴제).
- `invokeAi` 단일 진입점 · `prisma` 핵심키 · Express schema 키 = ADR 없이 수정 금지.
- 디자인 = 킷 260529(accent #F05519 1개 · radius 0 · NanumHuman/Poppins).
