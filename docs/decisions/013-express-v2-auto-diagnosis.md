## 배경

ADR-011 (2026-04-27) 채택 후 Express Track 이 6개월 운영되며 3가지 문제가 드러났다.

### 1. 슬기님 피드백 — "사회공헌 vs 일반전략" 프레임 판별 부재

신한금융 사회공헌 사업 제안서 작업 중 정상훈 고문이 지적:

> *"플랫폼 경쟁 심화 → 수수료 차별화 한계 → 비즈니스 전략으로 AX 도입. 이건 신한 기획/전략팀 언어야. 사회공헌팀 언어가 아님."*

PM 이 작성한 1차본의 첫 5문장이 어느 부서로 끌고 갈지가 발주처 채택 여부를 결정한다. 우리 시스템의 Inspector 7 렌즈에는 이 진단이 없었다.

### 2. 3 채널 메커니즘 차이 무시

`ChannelPreset` 시드는 B2G/B2B/renewal 3건 존재. 그러나 코드에서는 "톤" 만 분리되고 슬롯·Inspector·UI 흐름은 단일.

| 차원 | B2G | B2B | renewal |
|---|---|---|---|
| 발주처 정체 | 공공기관 (담당관) | 기업 부서 (CSR/전략/영업/기술) | 전년 발주처 (관계 형성됨) |
| RFP 성격 | 공식 문서 명확 | 모호 (Concept Paper / 미팅) | 작년 수정판 |
| 평가 방식 | 정량 배점 + 평가위원 | 부서 정치 + 의사결정 라인 | 전년 성과 vs 올해 개선 |
| 핵심 자료 | RFP + 평가표 | 발주처 신년사·CSR 보고서 | 작년 운영 데이터 |
| 위험 신호 | "지정 목차 위반" | "프레임이 다른 부서로 끌고 감" | "작년 미흡점 미반영" |

### 3. UI/UX 분산 — "다음 1 액션" 부재

`/projects/[id]` 우측 사이드바가 5+ 카드 더미 (IMPACT Value Chain 다이어그램 / AI 리서치 / 흔한 실수 / 당선 레퍼런스 등). PM 이 "지금 뭘 해야 하나" 답을 못 얻음. 슬기님–새싹 슬랙 대화 패턴 (진단 → 우선순위 → 컨펌 → 실행) 과 정반대.

### 4. 토큰 절약 패러다임의 함정

ADR-011 의 "외부 LLM 위임으로 토큰 절약" 가설은 PM 부담을 늘렸다:
- 외부 LLM 카드 5건 → ChatGPT 왔다갔다 5번 → 30~45분 목표 깨짐
- 슬기님: *"무거운 리서치만 외부에서 하자. 너무 왔다갔다 번거롭잖아"*

## 결정

**Express 2.0 패러다임 전환** — AI 가 "콘텐츠 생성자" 가 아니라 **"오케스트레이터·자동 진단자"** 가 된다.

### 1. AI 자동 진단 4종 도입

PM 입력 없이 AI 가 매 턴 또는 1차본 조립 직전 자동 실행:

| 진단 | 모듈 | 토큰/회 | 트리거 |
|---|---|---|---|
| **채널 추론** | `channel-detector.ts` | ~1K | Express 진입 시 1회 |
| **프레임 진단** | `framing-inspector.ts` | ~2K | sections.* 슬롯 채워질 때마다 |
| **팩트체크** | `fact-checker.ts` | ~3K | 1차본 조립 직전 |
| **논리 흐름** | `logic-chain-checker.ts` (gate3-validation 확장) | ~3K | 1차본 조립 직전 |

총 추가 토큰: ~50K → ~50K (turn 횟수 ↓ + 자동 진단 ↑ 상쇄). 사용자 허락 영역.

### 2. 슬롯 추가 X — 12 슬롯 유지

PM 입력 부담을 늘리지 않는다. 추가 정보는 **AI 가 RFP + 발주처 정보 + UD 자산 + 외부 LLM 답에서 자동 추론**.

### 3. 채널 분기 — 슬롯이 아닌 Inspector·UI 차원

| 영역 | 분기 방식 |
|---|---|
| 슬롯 | 12개 공통 (분기 X) |
| Inspector 가중치 | 채널별 분기 ⭐ |
| 사이드바 UI | 채널별 분기 ⭐ |
| 자동 진단 lens | 채널별 분기 ⭐ |
| 1차본 톤 (기존) | 채널별 분기 ✅ 이미 있음 |

### 4. 외부 LLM 카드 — 5건 → 2~3건

기준: **"우리 AI 가 절대 모르는 것만"**

| 사용 | 우리 AI 가능? | 카드로 |
|---|---|---|
| 정부 통계 (실시간) | ❌ | ✅ |
| 발주처 공식 문서 (CEO 신년사, 지속가능경영보고서) | ❌ | ✅ |
| 시장 데이터 (배달앱 MAU 등) | ❌ | ✅ |
| 키 메시지 / 차별화 / 섹션 본문 / 평가위원 시뮬 | ✅ | ❌ |

### 5. 사이드바 — 행동 흐름 4 패널

```
1. 다음 1 액션 (critical 1개만 강조)
2. AI 자동 진단 결과 (pass / warn / fail)
3. 외부 LLM 카드 (2~3건만, Impact Value Chain stage 태깅)
4. 슬롯 진행률 (12/12)
```

### 6. 의사결정 컨펌 흐름 (메타 layer)

기존 slot filling 흐름에 4 마일스톤 컨펌 추가:

1. 채널 결정 후 (1 클릭)
2. 메인 솔루션 결정 후 (1 클릭)
3. 1차본 조립 직전 (1 클릭)
4. 검수 결과 후 (수정 영역 선택)

슬랙 새싹 패턴 흡수 — 계획 → 컨펌 → 실행 → 변경 내역.

## 채널별 메커니즘 명세

### B2G — 정량성·정확성·경쟁

**Inspector 가중치**:
- evalWeightAlignment 30% — 평가배점 정렬
- quantitativeEvidence 20% — 정량 증거
- rfpCompliance 20% — 지정 목차 등 준수
- differentiation 15%
- udAssetCitation 10%
- framingAlignment 5%

**사이드바 특화 카드**: 평가배점 100점 만점 시뮬 (섹션별 예상 점수)

**자동 진단 lens**: rfpCompliance / evalWeightCoverage

### B2B — 프레임·부서·인용

**Inspector 가중치**:
- framingAlignment 30% — 사회공헌 vs 일반전략
- officialDocCitation 20% — 발주처 공식 문서 인용
- departmentTone 15% — 부서 톤 (CSR/전략/영업/기술)
- internalContext 15% — 내부 정보 반영
- differentiation 10%
- quantitativeEvidence 10%

**사이드바 특화 카드**: 프레임 진단 (감지된 부서 / 목표 부서 / 수정 제안)

**자동 진단 lens**: framing / departmentTone

### renewal — 개선·연속

**Inspector 가중치**:
- improvementMapping 30% — 작년 미흡 → 올해 개선 매핑
- priorYearData 25%
- measurableChange 20%
- continuity 15%
- differentiation 10%

**사이드바 특화 카드**: 전년 vs 올해 매핑 표 (체크리스트)

**자동 진단 lens**: improvementCoverage / continuityCheck

## 점진 실행 로드맵 (Phase M)

### Phase M0 (반나절~1일) — 시작 ⭐
- ChannelDetector + 컨펌 카드
- FramingInspector (B2B 우선, B2G·renewal 후속)
- 사이드바 재설계 (외부 LLM 5→3 축소 + AI 자동 진단 패널)

- FactCheckLight (정규식 + AI 검증)
- LogicChainChecker 채널별 확장
- 의사결정 컨펌 흐름 (4 마일스톤)

### Phase M2 (3~5일)
- 채널별 Inspector 가중치 분기
- B2G 평가배점 시뮬
- renewal 개선 매핑 자동화

### Phase M3 (1주+)
- PPT 출력 (Sassac 연결 또는 pptxgenjs)
- 발주처 공식 문서 ingestion (PDF → 키워드 추출)

## 대안 검토

### 대안 1. 17 슬롯 (PM 입력 +5) — 기각

PM 부담 30분 → 50~70분 증가. ADR-011 의 "30~45분 1차본" 목표 깨짐.

### 대안 2. 외부 LLM 카드 5건 유지 — 기각

사용자 직접 요청: *"너무 왔다갔다 번거롭잖아"*. 2~3건으로 축소.

### 대안 3. 채널별 슬롯 셋 분기 — 기각

PM 이 채널마다 다른 입력 학습 부담. 슬롯은 공통 유지, 분기는 Inspector·UI·진단 차원에서만.

### 대안 4. 토큰 더 절약 (AI 호출 ↓) — 기각

사용자 직접 허락: *"토큰은 좀 소모되어도 되는데, 무거운 리서치만 외부에서 하자"*. AI 가 더 일함.

## 리스크

### R1. AI 자동 진단 false positive

프레임 진단이 "사회공헌 → 일반전략" 오판할 수 있음. PM 이 무시할 수 있도록 *항상* `[무시]` 버튼 제공.

### R2. 토큰 비용

1차본당 ~50K → 진단 추가로 ~60K. 월 100 프로젝트 기준 추가 비용 ~$15. 허용 범위.

### R3. 채널 추론 오판

하이브리드 케이스 (카카오 지역협력 = B2G 라벨 + B2B 메커니즘) 가능. PM 컨펌 카드로 1 클릭 변경 가능.
