# Journey 로그 — 세션 단위 시행착오 기록

> **ADR 은 무엇을·왜 결정했나. Journey 는 어떻게 했나·뭘 틀렸나.** 둘 다 필요 —
> ADR 만 있으면 시행착오가 사라지고, Journey 만 있으면 결정의 권위가 사라진다.

## 룰
- **세션당 1건.** 파일명 `YYYY-MM-DD-<slug>.md`. 같은 날 여러 세션은 slug 로 구분.
- **추가 only. 과거 삭제 금지.** 사실이 바뀌면 새 엔트리.
- **HANDOFF 와 중복 OK** — Journey 는 영구, HANDOFF 는 라이브(덮어쓰기).

## 골격 (복사용)

```markdown
# YYYY-MM-DD · <한 줄 제목>

| 메타 | 값 |
|------|----|
| 메인 세션 | (모델/버전) |
| 작업 시간 | (대략) |
| 트리거 | (세션을 연 사용자 요청 — 가능하면 인용) |
| 관련 ADR | (있으면) |
| 관련 브리프 | (있으면) |
| 다음 세션 진입점 | HANDOFF.md 의 어느 섹션 |

## 한 일
(시간순 bullet · 메인 + 서브 에이전트 통합)

## 뭘 틀렸나 / 의외 발견
(다음 사람이 같은 함정 안 빠지게 — 교정과 함께)

## 결정한 것
(ADR 까지 안 갈 작은 결정. ADR 후보는 ADR 로.)

## 다음 세션이 알아야 할 것
(분기 가능 — "X 면 A, 아니면 B")

## 변경된 파일
NEW: ...
MODIFIED: ...
UNCHANGED (의도): ...   ← 코드 0건 등 무변경 증명
```

> 뎁스는 세션 무게에 비례. 골격은 불변, 본문은 늘어남.
