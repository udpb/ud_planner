# 2026-04-16 — 가이드북 리뷰 + A/B 재검토 철회

> 사용자가 별도 트랙으로 만든 `UD Proposal Planning Guidebook v1.0` 을 받아 반영 가치를 분석했는데, 내가 흥미로운 내용에 끌려 **Phase C 룰 엔진에 가이드북 사고 프레임을 섣불리 넣으려 했던** 실수. 사용자 한 질문("a와 B는 맞는 말인 것 같아?") 으로 철회. 게이트 책임이 어떻게 작동해야 하는지의 교본.

## 이날의 맥락
- Wave 2 (C4) 백그라운드 실행 대기 중
- 사용자가 가이드북 v1 HTML + 제작 로드맵 공유
- 사용자 의도: "기획자 OJT 배포용, 시스템과 별개, Practical"
- 다른 제안서 작성자의 스킬 체계 (proposal-planner/gen/visualize) 도 공유

## 내가 틀렸던 것

### 1. **가이드북 사고 프레임을 Gate 2 룰 엔진으로 변환 시도**
내가 제안한 A (3개 신규 룰):
- IMP-006 "그래서?" 도약 감지 — **부적합**: 의미 연결은 키워드 매칭 불가. Gate 3 영역.
- PROP-007 Output/Outcome 혼동 — **부적합**: "수료자 100명" 과 "역량 30% 향상" 은 문장 패턴만으로 구분 안 됨. Gate 3 영역.
- PROP-008 Section V 보너스 — 유효하지만 `PROP-005` 확장으로 가능, 별도 룰 과잉.

**근본 원인:** 룰 엔진의 설계 원칙 ("결정론적 + 단순 규칙") 을 내가 무시하고 가이드북의 **질적 사고 프레임**을 그대로 코드로 옮기려 함. 결과: false positive/negative 폭증할 룰들.

### 2. **WinningPattern 시드로 가이드북 요약문 사용 시도**
내 제안 B-1: NH 애그테크 · GS리테일 · 코오롱 프로보노 **Value Chain 요약** 을 시드 데이터로.

- **부적합:** 가이드북은 사람이 손으로 2차 가공한 요약. ADR-003 의 "원본 불변 보존" 원칙 위반.
- **진짜 맞는 방법:** 3건 모두 **원본 PDF 를 Ingestion UI 에 정규 업로드** → AI 추출 → Admin 승인. 같은 데이터라도 경로가 다름.

### 3. **오타 2건**
- "코스트롱 프로보즌" → **코오롱 프로보노** (정정)
- "NH 슈그포크" → **NH 애그테크** (정정)

검증 없이 웹페이지 텍스트를 인용. 브랜드 고유명사는 반드시 원문 확인했어야 함.

## 사용자 한 질문의 게이트 효과

**"a와 B는 너가 그대로 받아들이기보다 맞는 말인 것 같아?"**

이 질문이 없었으면 내가 C5 룰 엔진에 부적합한 3개 룰을 추가하고, Phase D1 브리프에 수동 시드 경로를 박아놨을 것. 한 번의 재검토 요청으로 설계 오염 차단.

## 내가 맞았던 것 (부분)

- **ChannelPreset 시드 3종 (Ch.12 카드) → Phase D2 유효** — 기존 `CHANNEL_TONE_PROMPT` 확장에 정확히 매핑
- **"흔한 실수 Top 7" (Ch.14) → pm-guide 콘텐츠 유효** — 데이터이고, 사람이 승인한 요약이라 그대로 인용 가능
- **코오롱 프로보노 사례 (2주 무한수정 → VC 후 1.5일)** — pm-guide "흔한 실수" 중 1위 항목으로 인용 가치 큼

## 가이드북 vs ud-ops 시스템 — 정체성 확정

**사용자 명시:** "가이드북이랑 지금 만드는 시스템은 별개야. 가이드북은 그냥 ojt처럼 배포용이야."

| | 가이드북 | ud-ops 시스템 |
|---|---------|-------------|
| 독자 | 기획자 (사람) | PM 작업 도구 |
| 형식 | HTML/PDF 배포본 | 웹 앱 |
| 역할 | 사고 프레임워크 학습 | 작업 실행·자산 축적 |
| 생명주기 | 주기적 업데이트 (분기/연) | 지속 배포 |

**두 개는 상보적**. 같은 가치(Impact Value Chain · 5대 역량)를 다루되, 가이드북은 **배우는 곳**, 시스템은 **실행하는 곳**.

## 신입에게 전할 말 (교육자료 씨앗)

**"설계 원칙 > 흥미로운 인풋"**
새 자료(가이드북·다른 팀 프로세스·외부 베스트 프랙티스)가 들어오면 반사적으로 "어디 반영할까?" 생각하는 게 위험. **설계 원칙(여기선 Gate 2 룰의 결정론성)을 먼저 점검**하고, 맞지 않는 건 다른 레이어(Gate 3 AI · 자산 · 가이드북)로 분리 배치해야 함.

**"고유명사는 원문 확인"**
브랜드명·프로젝트명은 기억이나 추측으로 쓰지 말 것. 사용자가 공유한 원본에서 직접 복사.

**"한 번의 재검토 질문이 설계 한 단계를 구한다"**
사용자의 "맞는 말인 것 같아?" 는 피드백이 아니라 **게이트**. 이게 없었으면 틀린 코드가 Phase C 에 들어갔음.
