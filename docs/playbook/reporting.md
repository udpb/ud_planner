# 보고 포맷 — 5섹션

> 메인 → 사용자 보고 (세션 끝 / 주요 완료 / "어디까지 됐나"). 서브 → 메인 Return Format 도 동일 5섹션 (대칭).

---

## 5섹션 포맷

```markdown
## ✅ 한 일
- <구체적으로 — 파일:라인 · 명령 · 결과>

## ❌ 못한 일 / 보류
- <항목> — 사유: <왜 못 했는지 / 무엇이 필요한지>

## 🤔 결정한 것
- <메인이 사용자 확인 없이 판단한 것 — 사용자가 뒤집을 수 있도록 명시>

## 🔬 검증
- <typecheck · lint · check:manifest · e2e 결과>
- <육안 확인한 항목 / 검증 못 한 것 명시>

## ⚠️ 위험 신호 / 다음 진입점
- <다음 세션이 알아야 할 것 / 사용자가 결정해야 할 것 / 기술 부채>
```

---

## 원칙

**"신뢰는 투명성에서 나온다. 못 한 것을 못 했다고 말하라."**

금지:
- 형용사 자랑 ("훌륭하게 완료")
- 에러 숨기기
- 검증 안 한 것을 "동작함" 으로 주장
- 무한 TODO 추가
- 에이전트 출력 그대로 전달 (메인이 **큐레이션**)

---

## 보조 포맷

**중간 한 줄:** `[<단계 N/M>] <한 일 한 줄> · <검증 한 줄> · <다음 단계 한 줄>`

**결정 요청:** 사전 소화해서 제시 — `옵션 A(장/단) / 옵션 B(장/단) / 추천: A(이유) / 진행할까요?`. 절대 "어떻게 할까요?" 만 던지지 않기.
