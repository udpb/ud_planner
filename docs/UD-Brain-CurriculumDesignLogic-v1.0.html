CURRICULUM DESIGN LOGIC · v1.0 · 2026-06-12 · ADR-028 기반
From 143 Winning Proposals

커리큘럼 설계 로직 v1.0 —
당선작 143건이 가르쳐 준
설계 문법으로 짓는다

추측이 아니라 실측이다. 언더독스가 실제로 이긴 제안서 143건에서 운영 구조를 16축으로 추출한 결과, 프로그램 설계에는 반복되는 문법이 있었다 — 포맷은 5개 아키타입으로 수렴하고, 회차는 "이론 전반 → 실행 → 코칭 후반 → 공유 종반"의 아크를 그리며, 진단·팀빌딩·전담코칭은 한 묶음으로 움직인다. 이 문서는 그 발견을 정리하고, 브레인이 앞으로 커리큘럼 1차안을 어떤 결정 순서로 짓는지(D0~D7)를 정의한다.

01

무엇을 근거로 말하는가

코퍼스 = 수주 제안서 원문 148건 중 143건 추출 성공 (총 186만 자, 2022~2025). 모든 추출값에 원문 인용(evidence)과 신뢰도(confidence)가 붙어 있고, 원문에 없는 값은 추측 없이 null 처리했다 — 강의 분류 v5.4의 "[파악 불가]" 원칙.

읽기 전 주의  이 분포는 "언더독스가 이렇게 설계해서 이겼다"이지 "이렇게 해야 이긴다"가 아니다 — 패배 케이스가 코퍼스에 0건이라 당락 변별은 아직 학습 불가. 또한 제안서에 거의 안 적히는 축(시간대·일부 KPI)은 운영 실적 데이터로 역보강해야 한다. 세그먼트 표본: 청년·대학생 49 / 임직원 14 / 소상공인·상인 9 / 청소년 6 / 시니어 2.

05

"깊은 육성 패키지" — 진단·팀·코칭은 따로 결정하지 않는다

진단 투입 모드 (45건)

인원 중앙 24.5명 · 기간 6개월 · 팀 기반율 0.93 · 코칭 중앙 5회

DOGS(26건) 등 사전진단 → 팀빌딩 → 전담 코칭이 한 묶음. 소수 정예를 깊게 끌고 가는 사업의 표준 패키지.

미투입 모드 (98건)

인원 중앙 40명 · 기간 3.5개월 · 팀 기반율 0.54 · 코칭 중앙 2회

다수 대상 단기 확산형 — 진단 없이 넓게 가르치고 행사로 모은다.

설계 함의: "진단을 넣을까?"는 독립 질문이 아니다. 육성 깊이(소수·장기·밀착)를 택했는지가 먼저고, 택했다면 진단+팀빌딩+코칭 5회 내외가 세트로 따라온다. 사전진단 투입률 31%는 언더독스 차별화 자산(DOGS·ACTT)의 실제 사용 빈도이기도 하다.

06

설계 로직 D0~D7 — 브레인이 커리큘럼을 짓는 순서

위 발견을 결정 순서로 조립한 것. 각 단계는 근거 데이터(approved 되면 DesignRule)를 컨텍스트로 소비하고, 모든 결정 옆에 근거를 표시해 사람이 결과가 아니라 이유를 검수하게 한다.

D0

입력 확정 — ProgramProfile

채널·대상(단계×인구통계)·예산·기간·과업 구성을 RFP에서 확정. 여기까지는 기존 Step 1 그대로 — 새 로직의 입력값이 된다.

근거 연결: 추출된 143건의 profileSnapshot 과 같은 어휘 — 유사 사업 검색이 이 단계에서 바로 가능.
D1

아키타입 선택 — 회차표보다 먼저

A1 교육 강좌형 / A2 몰입 캠프형 / A3 장기 여정형 / A4 개별 보육형 / A5 행사·대행형 중 택1(또는 주·부 조합). 분기 신호: 과업에서 교육의 비중(A5 판별) → 대상이 팀인가 개별 사업체인가(A4 판별) → 기간·예산(2~3개월 압축 vs 7개월+ 여정 — 양봉 분포라 중간값 제안 금지) → 몰입 가능성(캠프 vs 주차).

근거: 섹션 03 클러스터. A5(42건)에 회차표를 강요하던 것이 기존 1차안 어색함의 주요 원인.
D2

육성 깊이 결정 — 진단·팀·코칭 패키지

"소수·장기·밀착"(깊은 육성)인지 "다수·단기·확산"인지 결정. 깊은 육성이면 사전진단(DOGS/ACTT) + 팀빌딩 + 코칭 회차의 패키지가 함께 들어간다.

근거: 진단투입 45건 = 24.5명·6개월·팀 0.93·코칭 5회 vs 미투입 98건 = 40명·3.5개월·팀 0.54·코칭 2회 — 묶음으로 움직임 확인.
D3

골격 수치 산출 — 회차 × 시간 × 코칭

아키타입 중앙값에서 출발해 예산·기간으로 보정: 총회차(전체 중앙 8회), 회당 시간(A1=3.2h, A2=8h, A3=3h), 코칭 회수 = 교육 회차 × 0.6 (깊은 육성이면 상향).

근거: 코칭/회차 비 중앙 0.6 (n=33) · 아키타입별 중앙값 표.
D4

시퀀스 배치 — 아크 문법 적용

이론은 전반(상대위치 ~0.4 이전), 실행·실습은 중반, 코칭은 실행 개시 후(0.66+ 에 과반), 중간공유회는 50% 지점, 데모데이·경진대회는 최종 회차. 네트워킹만 목적에 따라 자유 배치.

근거: 77건 시퀀스 정규화 — 데모데이 종반 33/34, 중간공유회 중반 27/28. 기존 검증 룰 R-001~004(이론 30% 상한·Action Week 필수 등)는 이 위에 그대로 작동.
D5

콘텐츠 믹스 — 본 교육과 VOD의 분업

본 교육 = 강연+워크숍·실습 패키지, 실무·도구 중심(107건), 난이도 상승 아크. 이론·사례·경험담·인터뷰형 콘텐츠는 VOD 사전학습으로 내보낸다 — 본 교육에서 경험담·인터뷰는 각 3건뿐이라는 게 실측 근거. VOD 후보는 강의 분류 v5.4 필터(청중×단계×영역×난이도×유효성=상시유효)로 자동 제안.

근거: contentMix 분포 + v5.4 전달형식 기본값 맵(경험담·인터뷰=입문~초급/공통 — 사전학습 적합성과 정합).
D6

약속 레이어 — 산출물·지원금·사후관리·KPI

참여자 산출물(표준: 사업계획서 > IR덱 > MVP)과 지원금 구조(상금/사업화지원금/후속연계), 사후관리(후속보육·투자연계·alumni — "없음"은 143건 중 18건뿐)를 커리큘럼과 함께 설계. 당선작은 교육만 약속하지 않는다.

근거: 산출물 사업계획서 63·IR덱 39·MVP 33 / 사후관리 부재는 12.6%뿐.
D7

검증 + 사람 검수 — 근거가 보이는 1차안

R-001~004 자동 검증 통과 후, 1차안의 모든 운영 결정 옆에 근거(DesignRule + 출처 제안서 + 유사 당선 사례)를 표시해 PM 에게 전달. PM 의 수용/거절이 규칙 신뢰도로 환류된다.

근거: 설계 문법 v0.2 의 검수 루프(P4~P5) — 검수 비용을 제안서마다가 아니라 규칙에 한 번 지불.
07

세그먼트별 검증된 기본값 — 가설이 아니라 판정 결과

v0.2의 가설 매트릭스를 실측으로 판정한 결과만 남긴 표. 판정 표시가 있는 칸만 설계 기본값으로 승격 후보이며, 나머지는 데이터 부족으로 가설 유지.
